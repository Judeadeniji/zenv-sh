---
title: Secrets
description: API endpoints for creating, reading, updating, and deleting encrypted secrets.
---

All secret data is encrypted client-side. The server stores and returns ciphertext only.

## Create a secret

```
POST /secrets
```

```json
{
  "project_id": "proj_...",
  "environment": "production",
  "name_hash": "base64-hmac-sha256-of-name",
  "ciphertext": "base64-aes-256-gcm-encrypted-value",
  "nonce": "base64-12-byte-nonce"
}
```

**Auth:** Session or Bearer token.

| Field | Type | Description |
|---|---|---|
| `project_id` | string | Project the secret belongs to |
| `environment` | string | `development`, `staging`, or `production` |
| `name_hash` | string | Base64-encoded HMAC-SHA256 hash of the secret name |
| `ciphertext` | string | Base64-encoded AES-256-GCM encrypted value |
| `nonce` | string | Base64-encoded 12-byte nonce used for encryption |

Returns `201 Created` with the created secret.

## Get a secret

```
GET /secrets/{nameHash}?project_id=...&environment=...
```

**Auth:** Session or Bearer token.

**Response:**

```json
{
  "id": "secret-uuid",
  "name_hash": "base64-hash",
  "ciphertext": "base64-ciphertext",
  "nonce": "base64-nonce",
  "version": 3,
  "created_at": "2026-01-15T10:30:00Z",
  "updated_at": "2026-03-01T14:00:00Z"
}
```

## List secrets

```
GET /secrets?project_id=...&environment=...
```

Returns all secrets for the given project and environment. Ciphertext is included â€” the client decrypts locally.

## Update a secret

```
PUT /secrets/{nameHash}?project_id=...&environment=...
```

```json
{
  "ciphertext": "base64-new-ciphertext",
  "nonce": "base64-new-nonce"
}
```

The version is auto-incremented. The previous version is preserved for rollback.

## Delete a secret

```
DELETE /secrets/{nameHash}?project_id=...&environment=...
```

Permanently deletes the secret and all its versions.

## Bulk fetch

Fetch multiple secrets in a single request:

```
POST /secrets/bulk
```

```json
{
  "project_id": "proj_...",
  "environment": "production",
  "name_hashes": ["base64-hash-1", "base64-hash-2"]
}
```

**Response:**

```json
{
  "secrets": [
    { "name_hash": "...", "ciphertext": "...", "nonce": "...", "version": 1 },
    { "name_hash": "...", "ciphertext": "...", "nonce": "...", "version": 2 }
  ]
}
```

## Version history

```
GET /secrets/{nameHash}/versions?project_id=...&environment=...
```

Returns all versions of a secret, ordered by version number.

## Rollback

Restore a secret to a previous version:

```
POST /secrets/{nameHash}/rollback?project_id=...&environment=...
```

```json
{
  "version": 1
}
```

This creates a new version with the old value â€” version history is append-only.

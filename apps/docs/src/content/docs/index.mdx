## Why zEnv?

<CardGrid>
  <Card title="Zero-Knowledge" icon="approve-check">
    All encryption and decryption happens on your machine. The server stores ciphertext it can never read.
  </Card>
  <Card title="CLI + SDK" icon="laptop">
    Manage secrets from the terminal with the Go CLI, or load them at runtime with the TypeScript SDK.
  </Card>
  <Card title="Self-Hostable" icon="setting">
    Run the entire stack on your own infrastructure with Docker Compose. No vendor lock-in.
  </Card>
  <Card title="Cross-Language Crypto" icon="random">
    The Amnesia engine ships in Go and TypeScript. Both produce byte-identical output, verified in CI.
  </Card>
</CardGrid>
